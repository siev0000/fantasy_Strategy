# 実装タスク（順次進行）

## 方針
- 1タスクずつ完了させてから次へ進む。
- 各タスクに「完了条件（DoD）」を置く。
- 先にゲームループを成立させ、後で深掘りする。

## Phase 1: ループ成立（最優先）

### Task 1: 村のターン収支（食料・資材）
- 状態: 完了（2026-03-04）
- 内容:
  - 村ごとに `生産`, `消費`, `在庫` を持つ。
  - ターン経過で在庫を更新する。
- DoD:
  - ターンボタンで在庫が増減する。
  - 在庫不足時に警告が出る。

### Task 2: 建設システム（村ベース）
- 状態: 完了（2026-03-04）
- 内容:
  - 村で建物を1つ選んで建設できる。
  - 建設完了で収支に補正がかかる。
- DoD:
  - 建設前後で収支数値が変わる。
  - 建設条件不足（資材不足）をブロックできる。

### Task 3: 敵データの作成
- 状態: 完了（2026-03-06）
- 内容:
  - 出現敵.Json
  - 出現地形事に出現する敵があるのでそれらを元にデータを作成する。
  - 地形の高度(5 + 5x(地形の高度Lv))を基準に-5までぶれて決定される。
  - 敵の設定でLv_Min~Lv_Max の範囲に条件が合えば、条件を満たし、条件を満たした敵からランダムで配置される。地形に強敵が入っているの時は地形Lv+1で算出し、そこに作成されたデータの種族を+3Lvしたステータスを反映させる。
  - データの作り方はキャラの時と同じ計算式、種族とクラスはデータにあるのでそれらを使う。クラスがないときは種族に全部のLvを割り当てる
  - 一旦は地形に敵データを入れるところまで。戦闘は行わない

## Phase 3: AIと拡張

### Task 7: 敵勢力AI（簡易）
- 内容:
  - ランダムで
  - 1ターンごとに「移動/建設/攻撃」を実行。
- DoD:
  - 5ターン以上、自動で勢力行動が継続する。

### Task 8: 勢力選択と開始条件
- 内容:
  - 開始時に勢力/種族を選び、初期値へ反映する。
- DoD:
  - 選択内容で初期村・ユニット・資源が変化する。

### Task 9: マルチプレイ部屋作成（ホスト/参加）
- 状態: 未着手
- 内容:
  - ホストが部屋を作成し、参加者は部屋に参加する。
  - ターン進行権限はホストのみ（進行の単一権限化）。
  - 参加者は最新のゲーム状態を受信して同期表示する。
- DoD:
  - 部屋作成→参加→同一マップが表示される。
  - ホストのターン進行が参加者画面に反映される。
  - 参加者側ではターン進行操作が無効化される。

### Task 10: セーブ/ロード（ホスト配布型）
- 状態: 着手中（2026-03-09）
- 内容:
  - ホストが進行状況をセーブデータとしてダウンロードできる。
  - セーブデータをロードして途中再開できる。
  - ロード時に「どの勢力を自分が担当するか」を割り振るUIを出す。
  - セーブデータに `version` を持たせ、将来の互換判定を可能にする。
  - 実行時Stateと保存DTOを分離し、`toSaveData / fromSaveData` 変換で扱う。
  - マップは `base(地形/高度)` と `dynamic(川/滝/溶岩/敵/視界)` を分けて保存する。
  - まずはオンデマンド取得（requestSaveSnapshot）で保存可能なJSONを出せる状態にする。
- DoD:
  - 同一セーブをロードしてターン/資源/村/ユニットが再現される。
  - 勢力割り振り後に各プレイヤーの操作対象が正しく切り替わる。
  - `version` 不一致時に警告を表示し、ロード可否を選べる。

## チュートリアル要否
- 結論: **必要**（少なくとも軽量版）。
- 理由:
  - 現在はモーダル/設定項目が多く、初見で目的が伝わりにくい。
  - 「最初に何を押せば進むか」を示すだけで離脱率が大きく下がる。

## 最小チュートリアル（MVP）
- 1. 「地形を生成」ボタンを案内
- 2. 「自キャラ」モーダルを開かせる
- 3. 「ターン経過」で収支が変わることを体験させる
- 4. 「次の目標（建設する）」を表示

## 実装メモ
- チュートリアルは固定フロー3〜4ステップで十分。
- 完全な長文説明より、画面上ハイライトと1行指示を優先する。
