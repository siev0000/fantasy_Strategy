# 実装タスク（順次進行）

## 方針
- 1タスクずつ完了させてから次へ進む。
- 各タスクに「完了条件（DoD）」を置く。
- 先にゲームループを成立させ、後で深掘りする。

## Phase 1: ループ成立（最優先）

### Task 1: 村のターン収支（食料・資材）
- 状態: 完了（2026-03-04）
- 内容:
  - 村ごとに `生産`, `消費`, `在庫` を持つ。
  - ターン経過で在庫を更新する。
- DoD:
  - ターンボタンで在庫が増減する。
  - 在庫不足時に警告が出る。

### Task 2: 建設システム（村ベース）
- 状態: 完了（2026-03-04）
- 内容:
  - 村で建物を1つ選んで建設できる。
  - 建設完了で収支に補正がかかる。
- DoD:
  - 建設前後で収支数値が変わる。
  - 建設条件不足（資材不足）をブロックできる。

### Task 3: ユニット行動（移動/AP）
- 状態: 完了（2026-03-04）
- 内容:
  - ユニットに `行動ポイント(AP)` を持たせる。
  - 移動にAPを消費し、ターンで回復する。
- DoD:
  - APが0なら移動不可。
  - 次ターンでAPが回復する。

## Phase 2: ゲーム性強化

### Task 4: 索敵と遭遇
- 状態: 次着手
- 内容:
  - 索敵範囲に敵が入ると「発見」状態にする。
  - 発見時にイベントモーダルまたは戦闘遷移を出す。
- DoD:
  - 発見ログが残る。
  - 発見後に戦闘に入れる。

### Task 5: 一般兵/ネームドの生成ルール固定
- 内容:
  - 生成比率・レベル帯・装備テーブルを定義化する。
- DoD:
  - 設定値変更で生成結果が追随する。

### Task 6: 装備変更UI
- 内容:
  - ユニットに装備を付け替え可能にする。
  - ステータスへ反映する。
- DoD:
  - 装備変更直後にステータス値が更新される。

## Phase 3: AIと拡張

### Task 7: 敵勢力AI（簡易）
- 内容:
  - 1ターンごとに「移動/建設/攻撃」のどれかを実行。
- DoD:
  - 5ターン以上、自動で勢力行動が継続する。

### Task 8: 勢力選択と開始条件
- 内容:
  - 開始時に勢力/種族を選び、初期値へ反映する。
- DoD:
  - 選択内容で初期村・ユニット・資源が変化する。

## チュートリアル要否
- 結論: **必要**（少なくとも軽量版）。
- 理由:
  - 現在はモーダル/設定項目が多く、初見で目的が伝わりにくい。
  - 「最初に何を押せば進むか」を示すだけで離脱率が大きく下がる。

## 最小チュートリアル（MVP）
- 1. 「地形を生成」ボタンを案内
- 2. 「自キャラ」モーダルを開かせる
- 3. 「ターン経過」で収支が変わることを体験させる
- 4. 「次の目標（建設する）」を表示

## 実装メモ
- チュートリアルは固定フロー3〜4ステップで十分。
- 完全な長文説明より、画面上ハイライトと1行指示を優先する。
