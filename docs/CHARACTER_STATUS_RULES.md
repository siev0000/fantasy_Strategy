# キャラ成長・ステータス決定ルール（実装メモ）

最終更新: 2026-03-05  
対象: `frontend/src/components/PhaserMapGeneratorPanel.vue` / `frontend/src/composables/unitStatusUtils.js`

## 1. 目的

- `技能`（数値）と `取得スキル`（名称リスト）を明確に分離する。
- レベルアップ時の種族/クラス成長ルールを固定し、仕様ブレを防ぐ。

## 2. 用語

- 総合Lv: ユニットの `level`（現在は 1〜10 の範囲で運用）。
- 種族Lv: 種族成長に使う内部レベル。
- クラスLv: クラス成長に使う内部レベル。
- 初期種族ボーナスLv: `5` 固定。計算には加算するが、内部Lv表示には含めない。
- 技能: `指揮/威圧/看破...` の数値系パラメータ。
- 取得スキル: `Skill1..Skill10`（または `スキル1..スキル10`）から得るスキル名。

## 3. レベル成長ルール

- 内部Lvの初期値は `種族Lv0 / クラスLv0` として扱う。
- ただしステータス/技能の計算時は、常に `種族Lv+5` として計算する。

## 3.1 非人族（人族以外）
- 総合Lvに応じて、種族LvとクラスLvを交互に上げる。
- ルール:
  - Lv1: 種族Lv+1
  - Lv2: クラスLv+1
  - Lv3: 種族Lv+1
  - Lv4: クラスLv+1
  - ...（以後交互）

例:
- 総合Lv5 -> 種族Lv3 / クラスLv2
- 総合Lv10 -> 種族Lv5 / クラスLv5

## 3.2 人族

- 種族Lvは上げない（常に0）。
- 総合LvぶんクラスLvだけ上げる。

例:
- 総合Lv5 -> 種族Lv0 / クラスLv5
- 総合Lv10 -> 種族Lv0 / クラスLv10

## 3.3 人族の第2クラス

- 総合Lv10以上で第2クラス選択可能。
- 第2クラスはメインクラスと同名は不可。
- 現在実装の取得量:
  - 第2クラスLvは `1` 扱い（= 第2クラスの `Skill1` を取得対象に含める）。
- 総合Lvが10未満に下がった場合は第2クラスを解除する。

## 4. ステータス計算ルール

対象ステータス: `HP / 攻撃 / 防御 / 魔力 / 精神 / 速度 / 命中`  
`SIZ` は別計算。

## 4.1 成長ベクトル

- 種族行の各ステータス値を `10` で割って四捨五入し、1レベル成長量にする。
- クラス行も同様に1レベル成長量を作る。

## 4.2 最終ステータス

- 最終値 = `種族成長量 × (初期種族ボーナスLv5 + 種族Lv)` + `クラス成長量 × クラスLv`
- 内部Lv表示は `種族Lv` のみ（初期ボーナス5は表示しない）。
- `SIZ` は:
  - 種族行の `SIZ` を優先
  - なければクラス行の `SIZ`
  - どちらもなければ `100`

## 5. 技能と取得スキルの分離ルール

## 5.1 技能（数値）

- 表示対象: `指揮/威圧/看破/早業/技術/隠密/索敵/農業/林業/漁業/工業/統治/交渉/魔法技術/信仰`
- 技能値の上昇方式はステータスと同じにする。
- 計算式:
  - `最終技能値 = 種族技能成長量 × (初期種族ボーナスLv5 + 種族Lv) + クラス技能成長量 × クラスLv`
- ここは「数値パラメータ」であり、`Skill1..10`（取得スキル名）とは別扱い。

## 5.2 取得スキル（名称）

- 種族行は `種族Lv` ぶん、クラス行は `クラスLv` ぶん、`SkillN/スキルN` を順に取得。
- 人族で第2クラスありの場合は、第2クラスも `Lv1` 分を追加取得。
- 重複スキル名はユニーク化して1件にまとめる。

## 6. キャラ作成時の初期値（現状）

- 統治者: 総合Lv `5〜10` のランダム。
- モブ: 総合Lv `5` 固定。
- どちらも作成時点で本メモの成長ルールを適用して `status / 技能 / 取得スキル` を生成。
- 初期計算では `種族Lv5分` を加算するが、内部Lv表示は `種族Lv0` 扱いとする。

## 7. テスト運用

- テストON時のみ、自キャラ画面で手動レベル変更を許可。
- `Lv-1 / Lv+1` で再計算を即時反映。
- 人族かつLv10以上で第2クラス設定UIを表示して検証可能。

## 8. 実装確認チェックリスト

- `技能` 欄は数値のみ表示される。
- `技能` はステータスと同じ式で増減する。
- `取得スキル` 欄は `Skill1..N` ベースで増減する。
- 自キャラ表示画面（自キャラ一覧）で各キャラの `取得スキル` 要約が確認できる。
- 初期時点で `種族Lv5分` が計算に乗るが、表示上の内部Lvは `種族Lv0` のまま。
- 非人族Lvアップで種族Lv/クラスLvが交互に増える。
- 人族LvアップでクラスLvのみ増える。
- 人族Lv10で第2クラスが設定でき、取得スキルに反映される。
- Lvを10未満に下げると第2クラスが解除される。
